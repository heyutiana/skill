# Fill-in Formulas for Paper Reading Notes

## Section Templates

### One-sentence summary
```
Using [method A + method B], solving [problem], achieving [result on dataset].
```

### Problems to solve
Ask yourself: "What did previous methods fail at?"

Common types:
- Low accuracy / low efficiency / poor generalization
- Missing ability (memory, reasoning, perception)
- Heavy dependency on annotations / compute

### Core method (per module)
```
What it does = [input] -> [processing] -> [output]
Why needed = without it -> [what problem occurs]
```

### Key experimental results
```
Best result = [dataset] on [metric]=[value]
Ablation finding = removing [module X] causes [drop], meaning [module X] is key
Key parameter = optimal [K=?], reason in one sentence
```

### Core innovation
```
Previous methods = [how they do it], but have [what problem]
This paper = [how it does it], through [what mechanism] solving it
```

### Weaknesses (four angles)
1. Depends on what? (model, data, hardware)
2. Tested where? (simulation vs real)
3. What is not done? (future work)
4. What assumption is too ideal?

### Inspiration
```
This paper tells me: [core idea]
I can use this on: [my problem]
Specifically: [one sentence]
```

### Concept explanation
```
[term] = [plain language explanation], like [everyday example]
```

### Robot perspective
Describe the algorithm as if you are the robot:
```
I am a robot. My task is [goal].
First, I [step 1].
Then, I [step 2].
...
Finally, I [complete].
```

This tests whether you truly understand the process flow.
